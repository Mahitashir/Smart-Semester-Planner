                               Smart Semester Planner
Project Overview
The Smart Semester Planner is an advanced, web-based relational database management system designed to optimize university course registration and academic progression. Built on a centralized, unified role-based architecture, the platform securely isolates functionalities between authorized administrators and students while eliminating redundant data storage. It bridges the gap between administrative curriculum control and student-facing academic planning, transforming a traditional, static course catalog into an intelligent, interactive dependency graph.
Beyond standard data storage, the core of the application is driven by complex algorithmic logic, automated degree auditing, and strict referential integrity protocols. The system actively cross-references a student’s historical performance and departmental risk factors to generate mathematically load-balanced semester schedules. Working in tandem with this recommendation engine is a dynamic categorical degree auditor, which aggregates completed courses against strict university constraints to proactively identify and resolve specific syllabus deficits. Simultaneously, the platform shields the master catalog and user data through recursive application-level dependency managers and normalized SQL restriction constraints. This multi-layered protection ensures that neither a student nor an administrator can create logical anomalies, guaranteeing absolute academic and structural data integrity across the entire ecosystem.

1. Smart Semester Load Balancing Instead of just throwing random classes at a student, the system actually thinks about their workload. It looks at their overall CGPA, how heavy they want their next semester to be (Low, Moderate, High), and the subjective difficulty of the courses. It then builds a balanced schedule so they don't burn out while trying to graduate on time.
2. Strict Prerequisite Validation & Recursive Dependency Management The database physically prevents students from jumping into advanced classes they aren't ready for by cross-referencing their academic history with the university's complex prerequisite graph. Not only does the system lock down higher-level courses until foundational ones are passed, but it also features a recursive dependency manager. If a student removes a previously completed foundational course (like CSE110), the system automatically triggers a cascading chain reaction, intelligently wiping any advanced dependent courses (like CSE111 or CSE220) from their profile to maintain absolute academic integrity.
3. Department-Specific Performance & Risk Analytics A student might be great at English but struggle in Math. Instead of just looking at a generic CGPA, our database groups and averages a student's past grades by specific departments. If the system sees a historical weakness in a specific area (like the math department), it flags that as an academic risk when recommending heavy courses in that field.
4. Intelligent Course Retake Management We built custom logic to handle grade improvements. If a student wants to retake a course, the system treats it differently than a brand-new class. It verifies their previous grade, gives the retake priority in the schedule generation, and adjusts their remaining credit limits for the semester automatically.
5. Failsafe Database Integrity & Prerequisite Protection The backend uses strict Third Normal Form (3NF) rules to protect the university's curriculum data. We specifically engineered a restriction constraint: an Admin cannot delete a foundational course (like CSE110) if an advanced course (like CSE111) currently relies on it as a prerequisite. This acts as a safeguard, preventing broken dependency chains and ensuring the catalog never has missing links.
6. Automated Categorical Degree Auditing & Deficit Resolution Moving beyond simple credit summation, the database performs a dynamic, category-by-category degree audit. Utilizing advanced SQL grouping and multi-table joins, the system evaluates completed courses against strict structural constraints, sorting credits into predefined buckets like Major Core, Free Electives, and General Education. The auditor then cross-references these grouped aggregates with the master curriculum to calculate precise categorical deficits. Instead of merely flagging a missing requirement, the module seamlessly passes this state data to the recommendation engine, which proactively prioritizes specific courses (e.g., a missing Humanities class) to ensure the student remains perfectly on track for graduation.
